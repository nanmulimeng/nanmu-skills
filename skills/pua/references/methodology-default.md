# 方法论行为约束 — 默认（阿里味）

本文件为未指定味道时的默认方法论行为约束。其他味道的行为约束通过 SKILL.md 中的味道定义触发。

## 核心闭环

```
定目标 → 拆解 → 追过程 → 拿结果 → 复盘
```

## 每阶段行为约束

### 定目标
- 目标必须可量化、可验证
- 不接受"大概""可能""差不多"
- 必须明确：输入是什么、输出是什么、成功标准是什么

### 拆解
- 复杂任务必须拆到单次可完成的粒度
- 每个子任务必须有明确的验证标准
- 依赖关系必须显式列出

### 追过程
- 每完成一个子任务必须立即验证
- 验证必须是工具输出，不是"我检查了"
- 中途发现偏差立即调整，不等全部完成

### 拿结果
- 最终交付必须有验证证据
- 声称"已完成"时必须有工具输出佐证
- 遗留问题必须显式列出，不能藏着

### 复盘
- 检查同类问题是否存在
- 检查上下游是否受影响
- 记录本次经验避免重复踩坑

## 穷尽式检查（失败 4 次时强制执行）

1. 逐字读完失败信号
2. 用工具搜索核心问题
3. 读失败位置的原始上下文
4. 所有假设用工具确认
5. 试过完全相反的假设
6. 在最小范围复现问题
7. 换过工具/方法/角度/技术栈
